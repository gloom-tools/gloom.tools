Features

Real-time event processing

Subscribes to Slack's message.channels and message.groups events via the Events API. Every message processed in-memory, no batch jobs, no polling. Sub-200ms detection latency.

40+ compiled regex patterns

AWS, GCP, Azure, GitHub, GitLab, Slack, Stripe, OpenAI, Anthropic, Telegram, Twilio, SendGrid, SSNs, credit cards, private keys, JWTs, database URIs, and more. Add custom patterns via GitHub sync or the Slack config modal.

Docker-native deployment

Single container, no dependencies. Ships as gloom/psst:latest on Docker Hub. Helm chart available for k8s. SQLite for storage (or Postgres if you prefer). Your data never leaves your infrastructure.

Configurable action pipeline

Thread reply, DM the user, post to alert channel, auto-delete, or just log. Configure per-severity via config.yaml or the Slack App Home UI. Webhook support for piping events to your SIEM.

SQLite audit trail

Every detection logged with full context: who, what, when, which channel, pattern matched, action taken. Query it directly, pipe it to your data warehouse, or use the built-in App Home dashboard.

Friendly, not scary

No shame, no blame. psst gives people a quiet heads up and helps them fix it. Thread replies are visible but gentle. DMs are educational. Security culture, not security theater.

How it works

01

Install

Create the Slack app from our manifest (one click), grab your bot token, and run the container. Five minutes from zero to monitoring.

docker run -d \ -e SLACK_BOT_TOKEN=xoxb-... \ -e SLACK_APP_TOKEN=xapp-... \ -v psst-data:/data \ gloom/psst:latest
02

Configure

Tune patterns, severities, and actions via config.yaml or the Slack App Home. Set your alert channel, enable auto-delete for critical secrets, adjust sensitivity per category.

# config.yaml alert_channel: "#security-alerts" actions: critical: [delete, dm, alert] high: [reply, dm, alert] medium: [reply] low: [log]
03

Relax

psst watches every channel 24/7. When a secret slips, it handles it quietly. Your security team gets the audit trail, your engineers get a friendly heads up. Use /psst status anytime.
